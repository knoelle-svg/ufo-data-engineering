# UFO Sightings Data Engineering Exercise

## Overview
This project demonstrates a simple, reproducible data engineering workflow using a public, real‑world dataset. The focus is on clarity, correctness, and reviewer usability rather than over‑engineering.

The pipeline includes:
- Programmatic download of a public dataset
- Relational modeling and ingestion into SQLite
- Analytical querying using SQL and Python
- Generation of tables and lightweight visual artifacts suitable for downstream BI tools (e.g., Power BI)

Execution is fully reproducible and designed to run end‑to‑end with a single command.

---

## Data Source

The dataset used in this project is the **TidyTuesday UFO Sightings dataset (2019‑06‑25)**, which contains over 80,000 reported UFO sightings worldwide.

- **Source repository:** https://github.com/rfordatascience/tidytuesday/tree/main/data/2019/2019-06-25
- **Raw CSV download:** https://raw.githubusercontent.com/rfordatascience/tidytuesday/main/data/2019/2019-06-25/ufo_sightings.csv
- **Approximate raw size:** ~14 MB (uncompressed)

This dataset was selected because it is publicly accessible without authentication, large enough to support meaningful analytical exploration, and representative of noisy, human‑reported real‑world data commonly encountered in operational analytics.

---

## Schema Design

The database schema is designed to support efficient analytical queries while remaining transparent and easy to reason about. The schema is documented visually using an Entity–Relationship diagram and applied programmatically during ingestion.

Key design choices include:
- Use of a surrogate primary key, since the source data does not include a unique identifier
- Storage of string attributes (e.g., city, state, country, shape) as `TEXT`
- Storage of encounter duration values as numeric values
- Preservation of date values in string form to allow flexible parsing

Indexes are added to commonly queried fields such as `state`, `country`, `shape`, and `date_time` to improve analytical performance.

---

## Ingestion Pipeline

The ingestion process is implemented in `ingest.py` and is fully reproducible. On each run, the script:

1. Downloads the raw CSV dataset
2. Creates the SQLite database
3. Loads all records into a single `ufo_sightings` table

Because the dataset is moderate in size (<100k rows), it is ingested in a single pass for clarity and reliability.

The generated SQLite database file (`ufo.db`) is excluded from version control and can be regenerated at any time.

## Analysis Approach

Analysis is performed in `analysis.py` using a combination of SQL queries and Python (pandas) analysis.

While ingestion preserves all global records, several analytical questions intentionally focus on U.S. sightings due to higher data completeness and more consistent geographic identifiers. Across all analyses, the emphasis is on **patterns in human reporting behavior**, not interpretation of the reported events themselves.

---

## Analysis

### Sightings by Decade
Reported sightings are grouped by decade to examine long‑term reporting patterns. Results show a sharp increase beginning in the late 20th century, particularly after the 1990s.

This trend is interpreted as a function of population growth, media exposure, internet access, and reduced friction in reporting rather than changes in underlying phenomena.

---

### Geographic Concentration of Sightings
U.S. sightings are ranked by state and evaluated cumulatively. The top five states account for roughly 35–40% of all U.S. reports, indicating moderate geographic concentration rather than uniform distribution.

This pattern is consistent with population density effects and regional differences in reporting behavior.

---

### Encounter Duration and Reporting Behavior
Encounter durations are highly right‑skewed: most reported sightings are brief, while a small number of extreme values inflate average durations.

Key observations include:
- Median reported duration of approximately three minutes
- A very long tail extending to several hours

Encounter duration is used here as a **quantitative proxy for perception and recall**, acknowledging that free‑text sighting descriptions themselves were not directly analyzed in this exercise.

---

## Plots (Generated by `analysis.py`)

As part of the analysis pipeline, `analysis.py` generates lightweight plots and saves them to the `docs/` directory. These artifacts support visual inspection and downstream BI use (e.g., Power BI dashboards).

- `docs/plot_sightings_by_decade.png` — long‑term reporting trends
- `docs/plot_top_states_pareto.png` — concentration of U.S. sightings by state with cumulative percentage
- `docs/plot_encounter_duration_bins.png` — distribution of reported encounter durations (binned)

Plots are generated programmatically and committed solely as review artifacts.

---

## Limitations & Bias

Reported sightings are heavily concentrated in a small number of primarily English‑speaking countries, with the United States accounting for the majority of submissions. This reflects reporting infrastructure, language accessibility, and the U.S.‑centric origins of civilian reporting systems rather than geographic prevalence of events.
